Le SIP

SIP pour Session Initiation Protocol.

  • Protocole standard et ouvert normalisé par l'IETF
  • VoIP mais aussi : IM, Video, Réalité virtuelle, jeux vidéo...
  • En France
    • 2 ISP : Free et SFR fournissent un compte SIP.
    • Il y a aussi des offres via Ippi et Keyyo Mobile.
  • Des providers internationaux.

Les éléments d'un réseau SIP

  • User agent
  • Registrar (optionnel)
  • Proxy (optionnel)

Exemples de topologies

  • P2P : User agent à user agent
  • Registrar + Proxy ouvert ex : sip2sip.info
  • Gateway vers le réseau PSTN ex : freephonie
  • SIP trunking ex : pbxes.org
  • Avoir son propre server / celui de l'entreprise

Quand utiliser le NDK

  • Quand on a une lib stable en C / C++
  • Une connaissance des toolchains GNU
  • Pour certaines optimisations

Comment utiliser le NDK

  • Sous linux c'est plus simple
  • En utilisant la toolchain du NDK
  • Si existante, en utilisant GNU configure de la lib, CC, sysroot etc... bref de la cross compilation
  • Au dessus :
    • Des bindings JNI fait main
    • Autogénéré : SWIG
  • Pour les accès aux api android, utiliser JNI dans l'autre sens mais attentions aux threads !

Le faux problème de la fragmentation

La fragmentation des versions d'Android n'est pas un vrai problème depuis la 1.6

  • Librairie de compatibilité, Action Bar Sherlock
  • Vraiment comprendre minSdk, targetSdk
  • Lint
  • Reflexion
  • Singleton / interface

Singleton / interface


public abstract class KeyguardWrapper {

    public static KeyguardWrapper getKeyguardManager(Activity activity) {
        KeyguardWrapper kw;
        if(Compatibility.isCompatible(5)) {
            kw = new Keyguard5();
        } else {
            kw = new Keyguard3();
        }
        kw.initActivity(activity);
        return kw;
    }
    
    public abstract void initActivity(Activity activity);
    public abstract void lock();
    public abstract void unlock();
}
